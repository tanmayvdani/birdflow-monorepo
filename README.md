# BirdFlow Monorepo: SkyTrails Example

This branch demonstrates how the **BirdFlow** ecosystem is used as the data engine for the **SkyTrails** project—a mobile visualization platform for bird migration.

## 🌉 SkyTrails Integration

The SkyTrails project consumes high-fidelity migration paths ("SkyTrails") generated by BirdFlow models. The integration follows a three-step pipeline:

### 1. Model Training & Fitting (`BirdFlowPy` & `BirdFlowPipeline`)
Models are trained using eBird abundance data and GPS tracking data to create a probabilistic transition matrix for a specific species (e.g., the Amur Falcon).

### 2. Payload Generation (`BirdFlowR`)
Using the `BirdFlowR` package, we generate synthetic migration routes and export them into a lightweight JSON format optimized for mobile devices.

**Example R Script (`generate_skytrails_payload.R`):**
```r
library(BirdFlowR)
library(jsonlite)

# Load the fitted model
bf <- load_model("amur_falcon_2024")

# Generate 500 synthetic migration routes
routes <- route(bf, n = 500, season = "postbreeding")

# Format for SkyTrails iOS App
payload <- list(
  species = "Amur Falcon",
  scientific_name = "Falco amurensis",
  paths = routes$paths,
  timesteps = routes$weeks
)

# Export to JSON
write_json(payload, "AmurFalcon_SkyTrails_Payload.json")
```

### 3. Mobile Visualization (SkyTrails App)
The SkyTrails iOS/Android app fetches these JSON payloads and renders the paths using a custom GPU-accelerated engine, allowing users to explore the journey of individual birds across continents in real-time.

## 📦 Components Used in this Example

- **[BirdFlowModels](./BirdFlowModels)**: Provides the pre-trained `amur_falcon` model.
- **[BirdFlowR](./BirdFlowR)**: Used for the `route()` function to generate the path data.
- **[BirdFlowPy](./BirdFlowPy)**: Used for the underlying entropy-regularized optimal transport calculations.

## 🚀 How to Run the Example

1. **Setup Environment:**
   ```bash
   git submodule update --init --recursive
   ```
2. **Generate Data:**
   Run the R script provided above to generate the `SkyTrails` payload.
3. **Deploy:**
   Upload the resulting JSON to the SkyTrails data bucket.
